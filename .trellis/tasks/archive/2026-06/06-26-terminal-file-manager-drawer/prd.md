# PRD: 终端页底部文件管理抽屉

## Goal

在终端页面增加一个类似 FinalShell 的文件管理入口：用户连接 SSH 后，可以通过终端页上的按钮从底部拉起文件管理面板，在不中断终端会话的情况下完成常用远程文件操作。

## User Value

- 用户不需要离开终端页就能管理当前 SSH 会话的远程文件。
- 文件上传、拖拽上传、删除、重命名、新建目录等高频操作可以和终端操作并行完成。
- 设置页提供开关，允许用户禁用终端内文件管理入口。

## Confirmed Facts From Code

- 终端页实现位于 `app/src/features/terminal/TerminalPage.tsx`，当前已有右侧书签抽屉和命令历史抽屉，但没有底部文件管理抽屉。
- 独立 SFTP 页实现位于 `app/src/features/sftp/SftpPage.tsx`，当前包含本地/远程双栏文件浏览、远程新建文件夹、重命名、删除、下载、上传、拖拽上传、快速编辑、收藏/最近路径、传输队列等能力。
- SFTP 状态集中在 `app/src/stores/sftp.ts`，终端关闭时会断开关联的 SFTP session 状态。
- SFTP 后端实现位于 `app/src-tauri/src/core/sftp.rs`，已有 list/mkdir/remove/rename/stat/read/write/transfer 等远程文件操作基础。
- 设置页实现位于 `app/src/features/settings/SettingsPage.tsx`，终端设置通过 `terminal.*` key 持久化，并通过 `terminal:settings-changed` 事件刷新终端页运行时设置。

## Requirements

- 终端页增加文件管理按钮，按钮受设置开关控制。
- 设置开关默认开启；用户可以在设置页关闭终端内文件管理入口。
- 按钮打开/收起一个从底部拉起的文件管理面板。
- 文件管理面板的 MVP 形态为当前远程目录管理，不做完整本地/远程双栏。
- 文件管理面板使用当前活动终端 tab 的 SSH/SFTP session，不要求用户重新选择会话。
- 面板至少支持远程目录浏览、新建文件夹、重命名、移动到目标目录、删除、下载、上传、拖拽上传、刷新、路径导航。
- 移动文件/目录进入 MVP：在远程文件列表中选择一个或多个条目，通过“移动到...”选择目标远程目录，移动时保留原文件名。
- 移动目标目录已存在同名条目时必须提示用户，不应静默覆盖。
- 抽屉首次打开时优先进入当前终端推断出的工作目录；无法推断或目录不可访问时，回退到该会话上次文件管理路径，再回退到 `/`。
- 抽屉支持拖拽调整高度；默认打开占终端内容区域约 40%，最小高度约 260px，最大不超过内容区域约 70%。
- 抽屉展开、收起、调整高度后，终端区域必须重新适配，避免 xterm 尺寸错位。
- 终端分屏时，文件管理抽屉跟随最后聚焦的终端面板；如果没有可靠聚焦记录，则回退到当前 active tab。
- 传输队列在抽屉内以紧凑可折叠状态条呈现：默认显示正在传输数量、进度/速度和展开入口，展开后显示传输列表与取消/清理已完成操作。
- 复用独立 SFTP 页快捷键，但只在抽屉打开且文件列表拥有焦点时生效：`F5` 刷新，`F2` 重命名，`Delete` 删除，`Ctrl/Cmd+Shift+N` 新建文件夹。
- 文件操作应优先复用独立 SFTP 模块已有逻辑、样式和后端 API，避免维护两套不一致的文件操作行为。
- 上传、下载、删除非空目录、覆盖文件等风险操作沿用现有确认、toast 和传输队列体验。
- 设置页增加“终端内文件管理”开关，保存后终端页立即生效。

## Acceptance Criteria

- 设置开启时，终端页工具区可见文件管理按钮；设置关闭时按钮不可见且无法打开抽屉。
- 有活动且 connected 的终端会话时，打开抽屉直接显示该会话的远程文件列表。
- 切换活动终端 tab 后，文件管理面板跟随当前 tab 的 session 切换或刷新。
- 面板内可新建文件夹、重命名、删除文件/目录、上传文件、拖拽上传、下载文件/目录，并显示对应反馈。
- 面板打开/关闭、终端分屏、右侧书签/历史抽屉之间没有明显遮挡或布局错乱。
- 关闭终端 session 后，关联文件管理状态正确清理，不继续显示失效 session 的远程文件。

## Likely Out Of Scope For MVP

- 权限/属主/分组编辑。
- 远程文件内容预览器或多标签编辑器；已有 Quick Edit 可继续复用。
- 跨服务器复制/同步。
- 传输断点续传。
