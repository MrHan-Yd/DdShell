# README 文档更新 PRD

## Goal

更新根目录 `README.md`，让项目首页文档反映 v0.2.5 已发布能力，尤其是终端 AI 助手、终端内文件管理、右侧工具栏、应用内更新和当前文档入口。

## User Value

- 新用户打开仓库即可看到当前版本的核心卖点，而不是停留在旧功能清单。
- README 能和 `docs/发布/v0.2.5-版本介绍.md` 中的发布内容保持一致。
- 项目能力介绍更完整，便于用户判断是否下载、使用或参与开发。

## Confirmed Facts

- 根目录存在 `README.md`，仓库中不存在 `app/README.md`。
- v0.2.5 已发布并 tag 到 `95f2b6a`。
- v0.2.5 重点包括终端 AI 助手、多模型切换、流式响应、推理信息、按服务器隔离历史问答、终端底部文件管理抽屉、右侧终端工具栏和应用内更新链路。
- README 当前功能清单尚未完整覆盖这些新增能力。

## Requirements

- 更新根目录 `README.md` 的项目亮点和已实现功能。
- 增加或调整终端 AI 助手相关说明，覆盖 provider 协议、多模型、流式响应、推理信息、命令方案/步骤、执行确认和历史记录。
- 增加终端内文件管理抽屉说明，并和独立 SFTP 页面能力区分清楚。
- 更新终端工具栏、命令宏、设置和应用内更新相关说明。
- 修正 README 中不存在的 `app/README.md` 链接，改指向实际存在的文档入口。
- 保持 README 为中文主文档，语气偏产品介绍，不写实现细节堆砌。

## Acceptance Criteria

- `README.md` 明确包含 v0.2.5 的主要用户可见能力。
- 文档中的内部链接指向存在的文件。
- 不引入未实现或已明确 out-of-scope 的能力，例如自动拉取模型列表。
- Markdown 格式检查通过。

## Out Of Scope

- 新增截图或 GIF。
- 重写 `docs/shell/` 或 `docs/app/` 文档体系。
- 改动代码或发布 workflow。

## Open Questions

- 无阻塞问题。按 v0.2.5 发布内容和现有 README 结构更新。
