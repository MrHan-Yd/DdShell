# 技术方案

## 方案

继续扩展 `CrLfNormalizer`，增加 `post_cr_buffer`：

- 当当前行已有文本并遇到单个 pending bare CR 后：
  - 如果下一个字节是 `[`，输出 `\r\n[`。
  - 如果下一个字节是 ESC，暂存 CR 后的 ANSI/CSI 序列，等控制序列结束后的第一个可见字节再判断。
  - 如果控制序列后的第一个可见字节是 `[`，输出 `\r\n` + 暂存控制序列 + `[`.
  - 否则输出原始 `\r` + 暂存字节 + 当前字节。

## 边界

- 只对“当前行已有文本 + bare CR + bracket prompt”做换行转换。
- ANSI 暂存只为登录 prompt 覆盖服务，不作为通用终端解析器。
- 如果遇到不完整 ANSI 序列，继续跨 chunk 暂存；flush 时原样输出，避免吞字节。

## 风险控制

- 非 prompt bare CR 仍原样输出。
- 只在 `[` prompt 特征出现时转为换行，避免把常规进度刷新全部拆行。
