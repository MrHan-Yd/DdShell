# PRD: 移动弹窗列表内容左距

## Goal

根据截图反馈，调整终端文件管理“移动到...”弹窗中目录列表区域的整体左侧间距，避免文件夹图标和列表内容贴近弹窗左侧。

## User Value

- 目录列表视觉更舒适，文件夹图标不会显得贴边。
- 列表内容和上方路径、搜索输入区域保持更一致的内容边界。
- 行 hover 背景也不会从弹窗最左侧开始，整体更像一个内嵌列表。

## Confirmed Facts From Code

- 移动目录弹窗实现位于 `app/src/features/terminal/RemoteDirectoryPicker.tsx`。
- 目录列表容器当前是 `border-t ... py-1`，没有左右 padding。
- 目录行当前通过按钮自身 `pl-7 pr-4` 推动图标，但按钮 hover 背景仍从列表容器最左侧开始。
- 用户截图红框指向的是整个目录列表内容区域，而不只是单个图标。

## Requirements

- 目录列表容器需要增加左右内边距，让列表内容整体内缩。
- 文件夹图标相对弹窗左侧要比当前更远。
- 目录行 hover 背景应在内缩区域内显示，不贴弹窗左边。
- 不改变目录加载、搜索、进入目录、移动确认逻辑。
- Classic 和 Aurora 下保持一致。

## Acceptance Criteria

- 打开“移动到...”弹窗时，红框所示目录列表内容整体离左边有明显距离。
- 文件夹图标不贴近弹窗左边。
- 行 hover 背景与列表内容左距一致。
- 前端构建通过。

## Out Of Scope

- 调整弹窗顶部、搜索框、底部按钮布局。
- 调整文件管理抽屉或 SFTP 页面列表。
