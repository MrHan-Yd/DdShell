# 实现青瓷主题

## 目标与价值

将已经确认的静态设计原型 `ui/ui-celadon/` 完整接入 DdShell 应用，形成可在设置中选择、可持久化、支持深色/浅色/跟随系统，并覆盖真实业务页面与交互状态的「青瓷 / Celadon」主题。

## 已确认事实

- 主题标识使用 `celadon`。
- 中文展示名使用二字名称「青瓷」，英文展示名使用 `Celadon`。
- 视觉来源为 `ui/ui-celadon/`，浅色以瓷白、淡青釉、墨青边线和少量釉金为主；暗色为窑夜墨青与低亮青釉高光。
- 用户要求遵循提交 `afef4487` 学到的完整主题接入方式，不能只加配色或只覆盖部分页面。
- 当前完整主题模板可参考 `rainlake`、`umbra`，它们均包含主题注册、设置入口、预览、i18n、CSS 入口和真实应用页面覆盖。

## 功能要求

1. 在应用主题注册表和类型守卫中加入 `celadon`，保证读取与持久化的 `ui.theme` 值有效。
2. 在设置页主题列表中加入青瓷卡片、主题预览和当前主题名称映射。
3. 增加中英文主题名称及说明文案。
4. 在应用入口导入青瓷主题 CSS。
5. 将原型视觉完整映射到 React 应用，覆盖：
   - 应用壳、标题栏、侧边栏、状态栏和通用控件
   - 设置页及其卡片、表单、弹层和主题选择状态
   - 连接管理页及主从面板、主机、表单和状态
   - 终端页、标签、终端区域、搜索和快捷操作
   - SFTP 页、目录树、文件列表、传输和弹层状态
6. 深色、浅色和跟随系统三种模式均可正常工作；青瓷保持浅色优先，但暗色模式必须是独立的窑夜视觉，而不是简单反相。
7. 青瓷 CSS 必须限定在 `[data-ui-theme="celadon"]` 下，不能污染其他主题。
8. 保留原型的瓷面内高光、细边和稀疏开片质感，纹理不得降低业务信息可读性。

## 验收标准

- 设置页能看到并选择「青瓷 / Celadon」，保存后主题持久化，重新加载仍有效。
- 设置页账户概览中的当前主题名称能正确显示青瓷。
- 青瓷主题在 dark、light、system 下均正确切换，且系统模式随系统配色解析。
- 设置、连接、终端、SFTP 及公共应用壳的关键选择器覆盖与现有完整主题模板保持一致。
- CSS 文件结构包含入口、tokens、base、components、layout、四个页面文件和 app overrides。
- 无其他主题名称、旧主题选择器或明显旧配色残留在青瓷 CSS 中。
- `pnpm --dir app build` 通过，相关测试通过，`git diff --check` 通过。

## 不在范围内

- 不修改静态原型目录的设计方向。
- 不重构现有主题系统或抽象所有主题 CSS。
- 不改变默认主题。
- 不推送远端。

## 开放问题

无。用户已明确要求直接实现已确认的青瓷原型。
