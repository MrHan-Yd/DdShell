# 修复连接名称过长布局

## 目标

修复连接管理详情头部在服务器名称过长时挤压连接状态和操作按钮、导致文字换行及布局错乱的问题。

## 已确认原因

- 问题位于 `ConnectionsPage.tsx` 的 `HostDetail` 头部。
- `.detail-head` 是横向 flex 布局，但 `.title-block` 没有 `min-width: 0` 与可收缩剩余空间约束。
- `.detail-title` 没有单行省略规则，状态徽章和右侧操作区也没有禁止收缩。
- 该 DOM 被所有主题共用，应在组件布局类上统一修复，而不是分别修改每个主题 CSS。

## 要求

- 长连接名称只占用可用剩余宽度，单行显示并以省略号截断。
- 完整名称可通过原生标题提示查看。
- “已连接”状态保持单行且完整可见。
- 收藏、编辑、删除按钮保持固定在右侧，不被名称挤压或换行。
- 短名称下布局和行为保持不变，修复对所有主题生效。

## 验收标准

- 使用明显超过详情头部宽度的中英文连接名称时，名称出现省略号。
- 状态徽章与三个操作按钮仍完整显示且垂直居中。
- `pnpm --dir app build` 通过。
- `git diff --check` 通过。

## 不在范围内

- 不改变连接名称数据或限制名称长度。
- 不修改连接列表项已有的截断行为。
- 不调整主题视觉风格。

## 开放问题

无。用户截图和现有代码足以确认预期。
