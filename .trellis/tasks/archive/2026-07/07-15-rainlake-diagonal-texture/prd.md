# 为夜雨霓湖主题添加斜线纹理

## 目标

在现有 `rainlake`（夜雨霓湖）主题中补充可辨识但不过度抢眼的斜线纹理，使实际应用界面与主题预览图的视觉语言一致，并参考 `draftgrid`（蓝栅）主题将纹理叠加到应用结构表面的做法。

## 已确认事实

- 夜雨霓湖主题预览图已经使用约 `105deg` 的稀疏斜线。
- 夜雨霓湖 token 中已定义非常轻微的应用背景斜线，但标题栏、侧栏、状态栏、主内容框和设置内容框的独立背景会削弱或遮挡纹理。
- 蓝栅主题在 `app-overrides.css` 中将格子纹理直接叠加到上述结构表面，因此实际界面中的主题元素更明显。
- 本次是现有主题的视觉补充，不新增主题 ID、不改布局、不改交互。

## 需求

1. 为夜雨霓湖的主要应用结构表面叠加稀疏斜线，方向和节奏与主题预览图保持一致。
2. 深色模式使用冷白/青色的低透明度线条；浅色模式使用深蓝/青色的低透明度线条。
3. 纹理需保留现有渐变、玻璃质感、阴影和主题配色，不影响文字与控件可读性。
4. 参考蓝栅的实现层级，但不复制正交格子；夜雨霓湖只使用单方向斜线，以维持主题差异。
5. 如有必要，微调现有全局主题斜线强度，使背景和结构表面形成一致但有层次的视觉。

## 验收标准

- 深色与浅色夜雨霓湖主题中，标题栏、侧栏、状态栏、主内容框和设置内容框均能看到克制的斜线纹理。
- 斜线不会覆盖边框、文字、图标或交互状态，也不会明显降低对比度。
- 其他主题的样式不受影响。
- `pnpm --dir app build` 通过。
- `git diff --check` 通过。

## 不在范围内

- 修改夜雨霓湖的主题名称、描述或预览布局。
- 修改 React DOM 结构。
- 给其他主题添加纹理。
- 调整业务功能或交互。

## 开放问题

无。用户提供的截图和“参考蓝栅主题格子”的要求已足够确定实现范围。
