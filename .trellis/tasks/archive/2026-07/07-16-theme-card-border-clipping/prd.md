# 修复主题选择卡片边框裁切

## 目标

修复设置页主题选择器在滚动到顶部或底部时，首行卡片上边框和末行卡片下边框显示不完整的问题，让选中态边框和阴影始终完整可见。

## 已确认事实

- 所有主题选项共用 `.theme-option-grid` 滚动容器和 `.theme-option-card` 卡片样式。
- 网格当前固定显示两行并启用纵向滚动。
- 滚动容器上下没有边缘内距，可视边界紧贴首尾卡片；卡片选中态的边框/阴影会被 `overflow-y: auto` 裁切。
- 问题应在公共网格样式中一次修复，不应为各主题分别添加补丁。

## 需求

- 为主题选择网格的顶部和底部保留足够的可视空间，避免卡片边框和选中态阴影被裁切。
- 同步调整网格最大高度，保持桌面端和窄屏下仍能完整展示两行卡片。
- 保留现有纵向滚动行为、右侧滚动条空间以及主题卡片尺寸。
- 修复对所有 UI 主题和明暗模式统一生效。

## 验收标准

- 滚动到最顶部时，首行两个主题卡片的上边框/选中态完整可见。
- 滚动到最底部时，末行两个主题卡片的下边框/选中态完整可见。
- 网格仍完整展示两行卡片，不因新增边缘空间裁掉第二行内容。
- 主题选择、滚动和响应式布局行为不变。
- 前端构建和相关测试通过，`git diff --check` 无格式错误。

## 不在范围内

- 不调整主题卡片内容、排序、名称或预览图。
- 不修改每套主题的视觉设计。
- 不改版本号或发布流程。

## 开放问题

无。
