# 星云尘埃 UI 设计说明

## 名称

这套主题命名为「星云尘埃」，英文副名为 `Nebula Dust`。

`Nebula Dust` 指深空里低亮、稀薄、颗粒化的星云尘埃：黑紫和暗蓝提供宇宙背景，粉橙只作为远处尘埃和恒星余辉出现。它不是彩色光带主题，也不追求极光式流动冲击，而是一套更低饱和、更安静的深空工具主题。

## 灵感来源

灵感来自长曝光深空照片里的尘埃云：主体几乎是黑紫和暗蓝，只有局部出现粉橙、雾紫和银蓝星点。尘埃不是大面积渐变，而是分散、低亮、带颗粒感的微光。

DdShell 是高频开发工具，所以「星云尘埃」把漂亮的宇宙感压低到背景和边缘。终端、列表、表格和编辑器仍以清晰对比为主，粉橙星尘只用于焦点、Logo、图表高光和少量装饰，避免和已有极光主题过于接近。

## 视觉原则

1. 深空先于色彩  
   大面积使用深空黑紫和暗蓝，彩色只做局部尘埃，不做满屏霓虹。

2. 星尘要颗粒化  
   首页和 Logo 使用稀疏星点、尘埃云和细小反光，避免连续大光带。

3. 粉橙要少  
   粉橙作为主记忆点，但只出现在焦点、状态、图表和 Logo 高光里。

4. 工具界面保持冷静  
   业务页面延续高密度桌面工具结构，色彩氛围服务于识别，不抢终端和文件列表的阅读。

## 色彩结构

- 深空：黑紫、暗蓝、低亮蓝黑
- 星云：暗雾紫、低饱和紫灰
- 尘埃：粉橙、玫瑰灰、弱金橙
- 反光：银蓝、星白
- 文本：冷白、蓝灰、低亮紫灰

## 适用场景

「星云尘埃」适合 DdShell 这类长期使用的代码工具：连接管理、终端、SFTP、监控、片段和工作流都可以共用。它比极光更低饱和、更克制，适合想要漂亮但不刺眼的深色视觉方向。
