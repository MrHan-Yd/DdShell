# 命令助手二期规划（候选）

以下为第二期可选增强点，按优先级建议排序，仅供评估。

## 1. 参数化片段与占位符跳转
- 价值：同一命令少改很多，适合带变量的命令模板。
- 方案：支持 `{{var}}` / `{{var:default}}`，回填后用 `Tab` 在占位符间跳转。
- 成本：中等，需要解析与光标定位。

## 2. 候选来源标识（# 号标签）
- 价值：区分“内置命令 vs 用户片段”，降低误选风险。
- 方案：候选项右侧显示 `#builtin` / `#user`，并可附 `#centos` / `#ubuntu` 等系统标签。
- 成本：低，主要为 UI 标识规则。

## 3. 片段置顶/固定（# 号标签）
- 价值：对极常用或高风险命令更可控，不完全受权重影响。
- 方案：支持 `#pin` 标签，带 `#pin` 的候选始终置顶，其余按权重排序。
- 成本：低，增加一个标记位即可。

## 4. 轻量过滤语法（# 号标签）
- 价值：候选多时快速缩小范围。
- 方案：允许 `//dep #docker` 或 `//dep #sys:ubuntu` 的过滤输入；基于现有标签/系统信息过滤。
- 成本：低到中，需解析额外 token。

## 5. 多行/危险命令的提示策略
- 价值：降低误操作心理压力，尤其在用户改成 Enter 回填时。
- 方案：检测换行或高风险命令特征时弹出轻提示（仅回填不执行，可配置关闭）。
- 成本：低，规则式检测即可。

## 6. 内置命令包版本化
- 价值：便于后续更新内置命令集，同时不影响用户片段与权重数据。
- 方案：内置命令集带版本号；更新时仅覆盖内置集合。
- 成本：低，偏数据规范。

## 7. 片段库导入/导出
- 价值：支持导出为可分享文件，多设备或团队快速复用片段。
- 方案：导出/导入本地文件（JSON/YAML 二选一，保持简单）；导入时可选择合并或覆盖。
- 成本：中等，需要校验与冲突处理。

## 技术规范（对齐全局技术栈）
- 桌面壳：Tauri 2（与全局版本策略保持一致）。
- 后端：Rust stable（依赖版本基线对齐 `docs/02-ARCH/TECH-STACK-DECISIONS.md`）。
- 前端：React 19 + TypeScript 5.8 + Vite 7（不引入新框架）。
- 状态：Zustand 5（沿用既有状态方案）。
- UI：Tailwind CSS 4 + shadcn/ui + Radix UI（不新增第三方 UI 库）。
- 动效：仅使用既有 Framer Motion 或现有 CSS 过渡，不引入新动画库。
- 终端组件：使用 `@xterm/xterm`，禁止新增已弃用的 `xterm` 包。
- 本地存储：SQLite（与现有 snippets/偏好数据一致）。
