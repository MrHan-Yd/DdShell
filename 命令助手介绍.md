# Command Center Snippets: Shell App 内置命令片段助手

这是 Shell App 的 Snippets/Command Center 子能力，面向 SSH 终端与命令中心的快速回填。该功能以 IDE 风格候选提示为交互形态，遵循 `docs/03-UX/COMMAND-CENTER-SPEC.md` 与 `docs/03-UX/TOKEN-INSERT-SPEC.md` 的“仅回填不执行”原则。

---

## 核心定位
- 场景：在 Shell App 内根据已输入前缀触发候选提示，并回填命令片段。
- 入口：在终端输入区通过 `//` 触发内联候选提示（不需要弹框输入）。
- 设置项（设置 -> 终端设置）：开启 `//` 快捷触发（默认关闭），例如输入 `//dep` 后弹出候选列表。
- 设置项（设置 -> 终端设置）：提供“重置偏好”入口，用于清除本地排序权重。
- 设置项（设置 -> 终端设置）：回填确认键，默认 `Tab` 回填；可选改为 `Enter`。
- 非目标：不在系统全局输入框拦截键盘输入；不做操作系统级全局助手。
- 约束：不支持“先输入命令再按快捷键弹出候选”；该能力仅由 `//` 内联触发。

## 交互要点（与现有规范对齐）
- 回填行为：默认 `Tab` 回填到当前光标位置，不自动执行；`↑/↓` 选择候选。
- 回填细节：选中并回填时，替换当前的 `//` 触发前缀为目标命令；`Esc` 取消时保留 `//` 原样。
- 焦点管理：插入后焦点回到终端输入区，不打断输入流。
- 状态反馈：加载/空/错误状态遵循 UI-SPEC。
- 安全提示：长文本回填提供确认（可关闭）。
- 误触退出：`Esc` 立即关闭候选并保留原始输入。
- 硬约束：所有操作仅回填，不执行；不会发送回车。

## 性能与存储
- 数据源：合并两类来源参与匹配：用户自定义片段（沿用现有命令片段功能与存储）；系统内置常用命令（覆盖 Linux 发行版常见差异，如 CentOS/Ubuntu）。系统内置命令不展示在命令片段页面，仅用于候选匹配。
- 索引：内存 Trie/前缀索引 + LRU 缓存热点内容。
- 预取：仅在候选进入视窗或选中前触发小批量预取。

## 系统匹配
- 连接建立后识别远端系统类型（如 Ubuntu/CentOS），候选提示优先匹配该系统对应的内置命令集合。
- 用户片段与系统命令共同参与排序与过滤，最终输出可回填候选列表。

## 隐私与权限
- 默认仅在 Shell App 内工作，不监听全局键盘。
- `//` 仅在终端输入区生效，不做系统级全局拦截。

---

## 快捷操作
| 动作 | 快捷键 |
| :--- | :--- |
| 终端内 `//` 触发（可选） | 终端输入区连续输入 `/` |
| 选择片段 | `↑` / `↓` |
| 确认回填 | `Tab`（默认） |
| 取消关闭 | `Esc` |

---

## 技术栈
* Backend: Rust (Tauri 2)
* Frontend: React + TypeScript + Tailwind CSS + shadcn/ui + Radix UI
* Storage: SQLite（snippets） + 内存索引 + LRU 缓存
