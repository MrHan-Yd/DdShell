# 替代 FinalShell 的跨平台客户端设计文档（个人版）

> 目标：打造一款仅个人使用、支持 Windows/macOS/Linux 的高质感远程连接工具，体验接近或优于 FinalShell，并重点追求类 macOS 的视觉与交互质感。

---

## 1. 产品目标与范围

### 1.1 产品定位
- 一个“高颜值 + 高效率 + 高稳定”的 SSH/SFTP 桌面客户端。
- 面向个人使用，优先保证流畅体验和可维护性，不做企业协作和复杂权限系统。

### 1.2 核心目标
- 跨平台一致体验：Windows/macOS/Linux。
- 极快启动与低内存占用（相较 Electron 方案更轻）。
- 终端体验专业：多标签、分屏、会话恢复、快捷键完整。
- 文件传输高效：SFTP 双栏、拖拽上传、断点续传（后续版本）。
- 安全可控：密钥管理、系统钥匙串、最小化明文存储。

### 1.3 非目标（当前阶段不做）
- 团队协作（共享连接、审批流）。
- 云端账号同步。
- 插件市场。

## 2. 技术栈选型（确定方案）

### 2.1 总体架构
- 桌面壳：`Tauri 2`
- 前端：`React + TypeScript + Vite`
- UI：`Tailwind CSS + shadcn/ui + Radix UI`
- 动效：`Framer Motion`
- 状态管理：`Zustand`
- 终端渲染：`xterm.js`（fit + webgl + unicode11 addons）
- 后端核心：`Rust`

### 2.2 Rust 侧建议依赖
- 异步运行时：`tokio`
- SSH 协议：`ssh2`（MVP）/ `russh`（中后期可评估迁移）
- SFTP：基于 `ssh2` 的 SFTP 接口
- 数据库：`sqlx + sqlite`（或 `rusqlite`）
- 系统钥匙串：`keyring`
- 序列化：`serde + serde_json + toml`
- 日志：`tracing + tracing-subscriber`
- 时间：`chrono`
- 唯一 ID：`uuid`

### 2.3 选型理由
- Tauri 允许“Rust 做能力层 + Web 做高质感界面”。
- React 生态成熟，shadcn/ui 适合快速搭建精致界面。
- xterm.js 是目前桌面端最成熟的终端组件之一，稳定且可深度定制。

## 3. 信息架构与导航设计

### 3.1 全局导航结构
- 左侧边栏（Primary Nav）
  - 连接（Connections）
  - 终端会话（Sessions）
  - 文件传输（Transfers）
  - 命令片段（Snippets）
  - 设置（Settings）
- 主工作区（Content）
  - 顶部标签栏（Tabs）
  - 中央内容区（终端或文件管理）
  - 底部状态栏（连接状态/编码/延迟）

### 3.2 页面层级
- 一级：主工作台（连接列表 + 标签内容）
- 二级：连接详情弹窗、快捷命令面板、全局设置面板
- 三级：高级设置（代理、端口转发、证书策略）

## 4. 精细化布局方案（重点）

### 4.1 主窗口布局（桌面端）
- 外层：`12 列网格 + 8pt 间距体系`
- 标题栏高度：`44px`
- 左侧栏宽度：`260px`（可收起到 `72px`）
- 顶部标签栏：`40px`
- 底部状态栏：`28px`
- 主内容区：自适应填充

### 4.2 连接工作台布局
- 左：连接树（分组/收藏/最近连接）
- 右：欢迎区 + 快速操作卡片
  - 新建连接
  - 导入 SSH 配置
  - 最近会话恢复

### 4.3 终端页布局
- 顶部：会话标签（支持拖拽排序）
- 中部：终端容器（支持单屏 / 水平分屏 / 垂直分屏）
- 右侧可选抽屉：会话信息（主机、延迟、编码、环境变量）
- 底部状态：连接状态、传输速率、CPU/内存（远程可选）

### 4.4 SFTP 双栏布局
- 左栏：本地目录树 + 文件列表
- 右栏：远程目录树 + 文件列表
- 中间：操作列（上传、下载、同步、删除、重命名）
- 底部：任务队列（进度、速度、失败重试）

### 4.5 响应式策略（桌面窗口变化）
- 宽度 > 1400：默认显示右侧信息抽屉
- 1000~1400：隐藏右侧抽屉，保留核心区域
- < 1000：左侧栏自动折叠，仅图标显示

## 5. 视觉规范（类 macOS 质感）

### 5.1 设计关键词
- 清透、克制、层次分明、细腻动效。

### 5.2 色彩系统
- 背景层（3层）：
  - Base：`#0F1115`（深色）/ `#F7F8FA`（浅色）
  - Surface：`#151922` / `#FFFFFF`
  - Elevated：`#1B2130` / `#FFFFFF`
- 品牌色：建议蓝紫系（例：`#5B8CFF`）
- 语义色：成功/警告/错误保持中等饱和，避免“荧光色”。

### 5.3 圆角、阴影、描边
- 控件圆角：`10px`
- 卡片圆角：`12px`
- 浮层圆角：`14px`
- 描边：1px 半透明边线（暗色下更明显）
- 阴影：两层阴影叠加（大模糊 + 小模糊）

### 5.4 字体与排版
- macOS：`SF Pro`
- Windows/Linux：`Inter`
- 中文回退：`PingFang SC / Microsoft YaHei / Noto Sans CJK`
- 字号梯度：12 / 13 / 14 / 16 / 20

### 5.5 动效参数
- 普通过渡：`120ms ease-out`
- 面板开合：`160~180ms cubic-bezier(0.2, 0.8, 0.2, 1)`
- 禁止夸张弹簧动画，保持“专业工具”风格。

### 5.6 毛玻璃与系统风格
- macOS：启用 vibrancy/透明材质（标题栏/侧栏）
- Windows：根据系统能力启用亚克力或退化为半透明纯色
- Linux：默认不依赖系统特效，使用一致的自绘层

## 6. 功能设计（MVP -> v2）

### 6.1 MVP 必备功能
1. 连接管理
   - 新建/编辑/删除连接
   - 分组与标签
   - 收藏与最近使用

2. SSH 终端
   - 多标签
   - 会话重连
   - 字体/主题切换
   - 快捷键映射（复制粘贴、清屏、分屏）

3. SFTP 文件传输
   - 双栏浏览
   - 上传/下载
   - 拖拽传输
   - 任务进度与失败重试

4. 密钥与凭据
   - 支持密码/私钥登录
   - 凭据进入系统钥匙串
   - 可选会话级解锁

5. Snippets
   - 常用命令片段
   - 一键发送到当前终端

### 6.2 v2 进阶功能
- SSH 隧道与端口转发管理
- 远程监控面板（CPU/内存/磁盘）
- 批量执行命令（多主机）
- 同步目录（单向/双向，带冲突策略）
- 会话录制与回放

## 7. 关键交互细节

### 7.1 新建连接流程
- `Cmd/Ctrl + N` 打开“新建连接”
- 必填项：名称、主机、端口、用户名、认证方式
- 高级项折叠：代理、跳板机、KeepAlive、编码
- 保存后可直接“连接并打开标签”

### 7.2 连接失败反馈
- 失败提示包含：错误类型、建议操作、重试按钮
- 错误分级：认证失败 / 网络超时 / 主机不可达 / 密钥不匹配

### 7.3 终端体验优化
- 首屏自动聚焦终端输入
- 粘贴多行命令时二次确认（可关闭）
- `Cmd/Ctrl + D` 关闭标签，二次保护可选

### 7.4 传输队列交互
- 任务状态：排队/进行中/完成/失败
- 支持暂停、继续、重试、清空完成项
- 失败任务保留日志，便于排查

## 8. 数据模型设计（简版）

### 8.1 主要实体
- `Host`
  - id, name, group_id, host, port, username, auth_type, key_ref, created_at, updated_at
- `HostGroup`
  - id, name, order_index
- `Session`
  - id, host_id, started_at, ended_at, status
- `Snippet`
  - id, title, content, tags, created_at
- `TransferTask`
  - id, direction, local_path, remote_path, progress, status, error_message

### 8.2 存储策略
- 非敏感配置：SQLite
- 敏感信息（密码/私钥口令）：系统钥匙串（仅存引用 ID 到数据库）

## 9. 模块架构设计

### 9.1 前端模块
- `app-shell`：窗口框架与全局布局
- `connection-center`：连接列表与编辑器
- `terminal-workspace`：标签、分屏、终端实例生命周期
- `sftp-panel`：文件浏览与传输队列 UI
- `settings-center`：主题、快捷键、行为配置

### 9.2 Rust 模块
- `core::ssh`：SSH 连接、会话管理、命令执行
- `core::sftp`：文件操作与队列
- `core::secret`：凭据读写（keyring）
- `core::store`：SQLite 数据访问
- `core::event`：向前端推送状态事件

### 9.3 前后端通信
- Tauri command：请求-响应（如“连接主机”、“读取目录”）
- Event：流式事件（如“传输进度更新”、“会话状态变化”）

## 10. 性能与稳定性目标

### 10.1 性能指标（MVP）
- 冷启动：< 2.5s
- 常驻内存：< 250MB（普通使用）
- 单会话输入延迟：体感实时（< 50ms）
- 1000 文件目录渲染：< 300ms

### 10.2 稳定性策略
- 连接层断线自动重连（指数退避）
- 传输任务持久化（异常退出后可恢复）
- 日志分级记录（error/warn/info/debug）

---

## 11. 安全设计要点

- 不在数据库明文保存密码与私钥口令。
- 首次连接主机时展示指纹确认（known_hosts 策略）。
- 高危命令可启用“发送前确认”。
- 本地日志默认脱敏（主机、用户名可掩码）。

## 15. 风险与规避

- Linux 桌面环境差异大：特效做降级策略。
- SSH 库行为差异：MVP 使用 `ssh2` 降低不确定性。
- UI 过度设计影响效率：以“信息密度 + 可读性”为优先。

---

## 16. 首版验收标准（DoD）

- 三平台可安装并稳定启动。
- 可管理 >= 50 个连接配置且检索流畅。
- 多标签 SSH 会话稳定可用。
- SFTP 上传/下载在常见网络下稳定。
- 敏感数据不落盘明文。
- UI 达到统一视觉规范（间距、动效、暗色主题一致）。
