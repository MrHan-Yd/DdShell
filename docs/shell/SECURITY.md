# Security Policy

## 1. 安全范围
- 客户端 SSH/SFTP 会话安全。
- 自部署同步服务安全。
- 凭据与敏感数据保护。

## 2. 数据保护策略
- 密码与私钥口令不得明文落盘。
- 客户端优先使用系统 keyring 存储敏感信息。
- 同步服务仅存脱敏配置与必要元数据。

## 3. 传输安全
- 强制 TLS 传输。
- 禁用不安全密码套件。
- 建议启用 HSTS 与最小 TLS 版本策略。

## 4. 主机验证策略
- 首次连接必须确认主机指纹。
- 指纹变化默认阻断并提示风险。
- 允许用户手动信任更新，但需要二次确认。

## 5. 漏洞披露
- 报告渠道：项目安全邮箱（维护者配置）。
- 报告内容：影响范围、复现步骤、风险等级、修复建议。
- 响应目标：
  - 72 小时内确认收悉。
  - 7 天内给出修复计划。

## 6. 安全发布流程
- 修复完成后先发布安全公告草案。
- 分配 CVE（如适用）。
- 发布补丁版本并附升级指引。
