# DdShell v0.3.1 发布

DdShell v0.3.1 是一个重要的 Windows 修复版本，基于 `v0.3.0..HEAD` 的实际提交整理。本次修复了 v0.3.0 的 Windows MSI 安装包无法安装（错误代码 2503）以及 MSI 安装用户应用内更新静默失败的问题，并将 Windows 应用内更新改为按安装类型自动选择最合适的流程。

## 亮点

### 修复 Windows MSI 安装报错 2503

v0.3.0 的 MSI 安装包在安装时会报出错误代码 2503，导致安装向导中途退出、无法完成安装。原因是安装包内嵌的一段安装路径保留脚本与 Windows Installer 的执行策略冲突。v0.3.1 移除了这段脚本，MSI 恢复为标准安装流程，双击即可正常完成安装。

### Windows 应用内更新按安装类型自动分流

应用现在会识别自身是通过 NSIS（.exe）还是 MSI 安装的，并采用各自最可靠的更新方式：

- **NSIS（.exe）用户**：保持原有体验，更新静默完成后确认重启即可，全程无需额外操作
- **MSI 用户**：更新包下载完成后，状态栏和设置页会出现"打开安装程序"提示；点击后唤起标准安装向导，按引导完成安装（可自行选择安装目录），完成后重新启动 DdShell 即可使用新版本

此前 MSI 用户的应用内更新会在下载完成后静默失败：应用退出但安装并未发生，重新打开仍是旧版本。新流程从机制上避免了这一问题。

## 其他改进

- 修复设置中主题选择卡片边框被裁切的显示问题

## 已知限制

- **v0.3.0 MSI 用户无法通过应用内更新升级到本版本**（旧版更新缺陷所致），请前往 [Releases 页面](https://github.com/MrHan-Yd/DdShell/releases/latest) 手动下载 `DdShell-v0.3.1-windows-x64.msi`（或 `.exe`）安装一次，之后即可正常使用应用内更新
- macOS 与 Linux 用户不受上述问题影响，正常升级即可
