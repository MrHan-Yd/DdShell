# DdShell v0.2.8 发布

DdShell v0.2.8 基于 `v0.2.7..HEAD` 的提交记录整理，是一次围绕 Windows 更新安全、终端交互效率和会话生命周期的维护发布。这个版本重点修复 Windows 应用内更新可能带来的安装目录和本地数据风险，同时给终端加入选中文本快捷操作，并让会话超时更贴近用户实际停留页面的行为。

## 亮点

### Windows 更新更稳妥

- Release manifest 区分 `windows-x86_64-nsis` 和 `windows-x86_64-msi`，不再发布会跨安装器类型 fallback 的宽泛 Windows 更新目标
- Windows Release 上传并校验 MSI updater 签名，避免 manifest 指向缺失签名的 MSI 产物
- NSIS 更新流程会备份并恢复安装目录内的 `shell.db`、`shell.db-wal`、`shell.db-shm`，降低更新卸载阶段导致本地连接数据丢失的风险
- MSI 更新固定 `upgradeCode`，加入 WiX 安装目录继承片段，并在 updater 运行时传入当前安装目录，提升非默认安装路径升级的稳定性

### 终端选中文本快捷操作

- 终端选中文本后显示紧凑的图标浮层，支持复制、复制并粘贴到当前光标处
- 浮层基于 xterm 公开 selection 坐标定位，修复第一行/顶部选区不显示或被裁切的问题
- 分屏终端中浮层只作用于当前选中的 pane，避免跨会话误操作

### 终端会话超时更符合使用习惯

- 终端页面处于前台时会定时刷新应用级活动时间，用户停留在终端页但暂时不输入不会触发会话过期
- 用户切到其他页面时刷新一次活动时间并停止前台保活，超时计时从切页时刻开始
- 新增独立的 `session_touch_activity` 命令，只刷新空闲计时，不发送 SSH 输入、不触发 resize、不混用延迟 ping

### 终端输入和界面细节修复

- 修复部分 SSH 终端回车显示 `^M`、命令不按预期提交的问题
- 修复连接页“连接中”加载态主按钮阴影覆盖到旁边“测试”按钮的问题
- 补充多套 UI 视觉方案和 Aurora 主题静态设计稿，为后续主题演进提供素材

## 其他改进

- 更新 Tauri updater、MSI/NSIS、终端会话 idle timeout、xterm selection overlay 和禁用态按钮阴影相关 Trellis 规范
- 记录 Windows MSI 安装目录保护、官方 updater manifest 平台 key、终端选择浮层定位等维护约束
- 归档终端选择快捷操作、Windows 更新修复、SSH 回车修复等任务，并补充开发 journal
- 发布版本号同步更新到 `0.2.8`

## 已知限制

- macOS 仍未购买 Apple Developer 签名时，首次打开可能需要在系统安全设置中手动放行，或执行发布页说明中的 `xattr -cr /Applications/DdShell.app`
- Linux 暂时继续使用 GitHub Releases 下载页作为应用内更新回退路径
- Windows 更新路径保护仍建议在发布后继续做 MSI/NSIS 非默认目录升级烟测，确认不同安装器来源的数据保留表现
