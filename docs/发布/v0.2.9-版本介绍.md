# DdShell v0.2.9 发布

DdShell v0.2.9 基于 `v0.2.8..HEAD` 的提交记录整理，是一次面向终端分屏、终端输入体验和 SFTP 大目录上传稳定性的维护发布。这个版本修复了分屏显示重复会话、终端文件管理与主文件管理上传进度不准确、整包上传完成后仍显示未完成等问题，并补齐密码输入显示开关，让用户可以按偏好选择本地输入回显行为。

## 亮点

### 终端分屏显示真实会话

- 分屏按钮现在会选择当前终端之外的另一个真实会话，不再把当前会话复制到第二个面板
- 只有一个可用目标时自动分屏到另一个会话，多个目标时弹出目标选择
- 已经分屏时再次点击水平或垂直分屏只切换方向，不改变第二屏目标
- 分屏快捷键和工具栏按钮复用同一套目标选择逻辑

### 终端输入和密码场景更可控

- 新增密码输入预测显示开关，默认保持可见输入回显
- 用户关闭后，密码提示场景会像普通终端一样输入但不显示内容
- 粘贴和预测回显增加敏感提示识别，降低密码输入时暴露灰色预测文本的风险
- 修复粘贴内容可能被截断或隐藏、看不到前段内容的问题

### SFTP 大目录上传进度更可靠

- 上传文件关闭阶段增加超时和远端大小校验，远端文件大小一致时按完成处理
- 完成态会强制把进度收敛到 100%，避免传完后仍显示未传完
- 主文件管理和终端内文件管理都按完整远端路径跟踪上传任务，避免目录包中同名 `index.ts`、`package.json` 等文件互相串进度
- 目录切换时只保留当前目录对应的上传占位，避免子目录上传误显示在父目录

### 终端工具栏和状态栏体验优化

- 简化终端工具栏图标，并调整水平/垂直分屏按钮顺序
- 终端 pane 延迟数值展示更稳定
- 状态栏隐藏后增加延迟退出，减少鼠标经过底部区域时的闪动

## 其他改进

- 更新 SFTP 上传状态跟踪、终端分屏实例复用、xterm 背景和选择浮层相关 Trellis 规范
- 补充终端真实分屏任务记录和上传进度修复的工程约束
- 继续保持 Windows updater、macOS updater 和 Linux fallback 的发布策略不变
- 发布版本号同步更新到 `0.2.9`

## 已知限制

- macOS 仍未购买 Apple Developer 签名时，首次打开可能需要在系统安全设置中手动放行，或执行发布页说明中的 `xattr -cr /Applications/DdShell.app`
- Linux 暂时继续使用 GitHub Releases 下载页作为应用内更新回退路径
- 大目录上传已修复前端状态跟踪和完成态收敛逻辑，仍建议发布后用真实远端服务器做一次大目录压力烟测
