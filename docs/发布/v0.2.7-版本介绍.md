# DdShell v0.2.7 发布

DdShell v0.2.7 基于 `v0.2.6..HEAD` 的提交记录整理，是一次安全基础、终端稳定性和文件管理体验更新。这个版本重点降低原生权限与本地资源暴露面，修复终端登录提示、文件管理覆盖层和远端 resize 之间的多类显示干扰，并让“无操作超时”按用户行为而不是系统心跳来判断。

## 亮点

### 安全基础收敛

- SSH 密码和 AI provider API key 接入系统钥匙串存储，保留本地 AES-GCM 加密回退，并支持旧本地密文懒迁移
- 终端背景图片改为导入到应用数据目录后再通过受限 asset scope 渲染，不再依赖宽泛本地文件访问
- Tauri capability 收窄到主窗口和 Quick Edit 实际需要的权限，减少不必要的插件授权
- 前端剪贴板访问统一走 Tauri 优先 helper，平台判断统一走共享 helper，避免散落的浏览器兼容字符串分支

### 终端登录和文件管理更稳定

- 修复登录提示、Last login、失败登录提示与 shell prompt 混排/覆盖的问题
- 抑制终端启动阶段和文件管理过渡阶段的远端 resize 副作用，减少 prompt 重绘、重复刷新和残片
- 文件管理打开、关闭和拖拽高度时尽量使用本地布局适配，不再频繁触发远端 PTY resize
- 修复重连遮罩下活动终端边框缺失、终端底部黑边等边界显示问题

### 文件管理和 SFTP 细节优化

- 调整 SFTP 文件管理页面两侧列表与边框的距离，收藏/路径下拉不再贴边
- 新建文件夹输入框补充过渡动画和更自然的样式
- 文件管理覆盖层减少对终端内容的闪烁干扰，关闭按钮入口统一，视觉状态更稳定

### 会话无操作超时按用户行为生效

- 新增后端应用层 idle watchdog，`session.keepAlive` 不再依赖 SSH 库的网络空闲判断
- 用户输入、resize、SFTP 操作和传输进度会刷新活动时间
- SSH keepalive、远端输出和状态栏延迟 ping 不算用户操作，因此不会让 30 秒无操作超时无限续期

## 其他改进

- 补充后端、前端和发布相关 Trellis 规范，记录 SSH 登录 banner、终端 overlay 层级、session idle timeout、Tauri updater 和权限范围等维护约束
- 优化安全质量门和前端拆包，减少主包压力并提升后续维护可读性
- 发布文档补充规则：版本介绍必须依据上一版本 tag 到当前发布提交的实际提交记录和 diff 统计生成
- 发布版本号同步更新到 `0.2.7`

## 已知限制

- macOS 仍未购买 Apple Developer 签名时，首次打开可能需要在系统安全设置中手动放行，或执行发布页说明中的 `xattr -cr /Applications/DdShell.app`
- Linux 暂时继续使用 GitHub Releases 下载页作为应用内更新回退路径
- 会话无操作超时按新建连接读取当前设置；已经建立的旧连接不会因为之后修改设置而即时变更 timeout
