# DdShell v0.2.2 发布

DdShell v0.2.2 将 Predictive Echo 从 M0 推进到 M1：高延迟 SSH 链路下的预测回显现在默认开启，但仍保留“实验”标识与关闭入口，方便更多真实环境参与验证。同时，本次版本也补齐了应用内更新链路，状态栏检查更新后可尽量直接下载并打开当前平台的安装包。

## 亮点

### Predictive Echo 进入 M1（默认开启，仍标实验）

- 预测回显现在默认开启，新安装或尚未保存该设置的环境会直接启用
- “预测回显（实验）”标签保留，不提前按 M2 口径转正
- 首次按默认值启用时仍会展示一次性引导：如遇异常，可在设置中关闭
- 已覆盖阶段 2 的 shell 兼容性、Ctrl+U / Ctrl+W、SGR 着色、中文 IME、光标对账等实现

### 应用内更新可按平台下载并打开安装包

- 状态栏检查更新后，会优先为当前平台选择合适的 Release 资产
- 支持应用内下载并直接打开安装包，减少手动去 GitHub Releases 找包的步骤
- 无法确定目标安装包或本机不支持直接打开时，会自动回退到浏览器发布页

## 其他改进

- 发布版本号同步更新到 `0.2.2`
- Predictive Echo 阶段 2 进度文档已切换到 M1 发布口径，后续继续记录 dogfood 与外部反馈
- 本次发布范围基于 `v0.2.1..HEAD` 的未发布改动整理，包含 Predictive Echo 阶段 2 与最近的应用内更新能力

## 已知限制

- Predictive Echo 仍是实验特性；如出现屏幕错乱、回显异常或特定 shell/IME 不兼容，可在设置中关闭
- 切片 7/8/9 的真实环境 dogfood 与外部反馈收集仍会在 M1 期间继续进行
- Linux 等不支持 MVP 安装包直开的场景，会继续走浏览器发布页兜底

## 回滚预案

- 如果 Predictive Echo 在真实使用中暴露 critical bug，优先通过设置关闭；必要时后续补丁会将默认值改回 `false`，回退到 M0 口径
