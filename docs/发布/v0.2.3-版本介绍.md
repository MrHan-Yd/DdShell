# DdShell v0.2.3 发布

DdShell v0.2.3 重点完成双主题体验收敛：经典主题在保留自身颜色体系的同时，对齐极光主题的页面布局、密度和交互细节；连接管理、终端、文件管理和设置等高频页面在主题切换时更加稳定。本次版本也统一了应用 Logo 与打包图标，减少不同主题之间的品牌视觉差异。

## 亮点

### 经典主题对齐极光布局

- 连接管理、设置、菜单导航、终端和文件管理页面采用更一致的布局骨架和控件密度
- 经典主题继续保留自己的颜色 token，不直接套用极光主题的紫色视觉
- 修复连接管理点击“连接”进入连接中状态时详情区宽度收缩的问题

### 极光主题与经典主题切换更稳定

- 对齐菜单导航字体、间距和顶部区域高度，减少切换主题时的视觉跳动
- 修复极光设置页 tab 区域、搜索图标定位等回归问题
- 补齐经典主题终端 pane header 样式，避免主机名和连接状态错位

### 应用品牌资源统一

- 程序内 Logo、浏览器 favicon 和 Tauri 打包图标统一使用暗色极光 Logo
- 重新生成 macOS、Windows、Linux、iOS、Android 等 Tauri 图标资源

## 其他改进

- 文件管理页面补齐经典主题的双栏文件面板、传输队列、工具栏和会话切换按钮布局
- 命令宏、片段、监控和连接管理等页面继续跟进设计稿视觉对齐
- 空状态图标、上下文菜单、分段控制器、按钮和表单控件细节进一步统一
- 发布版本号同步更新到 `0.2.3`

## 已知限制

- macOS 包如果未配置 Apple Developer 签名与公证，首次打开仍可能需要在系统安全设置中手动放行，或按发布页说明执行 `xattr -cr /Applications/DdShell.app`
- 本次发布不调整 GitHub Actions release workflow，也不改变应用内更新、签名、公证和多平台打包策略

## 回滚预案

- 如新主题样式在真实环境中出现明显回归，可回退到 `v0.2.2` 安装包；后续补丁优先以主题作用域 CSS 修复，不改连接、终端或文件管理的数据模型
