# DdShell v0.2.6 发布

DdShell v0.2.6 是一次稳定性补丁发布，重点修复 macOS 应用内更新下载完成后打开安装包不可靠的问题，并补充发布链路检查记录。这个版本适合已经使用 v0.2.5 的用户升级，尤其是需要通过应用内更新获取后续版本的 macOS 用户。

## 亮点

### macOS 更新安装包打开更可靠

- 下载更新完成前会 flush、关闭并校验安装包文件，避免文件尚未完全落盘时 UI 已提示完成
- macOS 手动安装包打开改为调用系统 `/usr/bin/open`
- 如果系统打开安装包失败，会返回状态或 stderr 信息，便于 UI 回退到 GitHub Releases 下载页

### 更新失败场景更容易定位

- 下载文件缺失、不是普通文件或大小不匹配时，不再发送“下载完成”事件
- 安装包路径不存在或不是文件时，会直接返回错误
- 这些校验同时保护旧的手动下载打开链路，减少“点了没反应”的模糊状态

### 发布链路复核

- 复核 Windows 官方 updater 链路，确认当前主流程使用 Tauri `downloadAndInstall()`
- 复核 Windows updater 发布资产，确认 NSIS `.exe`、`.exe.sig`、`.msi` 和 `latest.json` 存在
- 确认 updater manifest 包含 `windows-x86_64-nsis` 和 `windows-x86_64` fallback，并指向已签名的 NSIS `.exe`

## 其他改进

- README 补充项目功能、快速开始、发布文档入口和当前版本介绍入口
- 补充 updater 相关任务记录，明确 macOS 手动打开安装包和 Windows updater 资产检查结论
- 发布版本号同步更新到 `0.2.6`

## 已知限制

- macOS 仍未购买 Apple Developer 签名时，首次打开可能需要在系统安全设置中手动放行，或执行发布页说明中的 `xattr -cr /Applications/DdShell.app`
- Linux 暂时继续使用 GitHub Releases 下载页作为应用内更新回退路径
