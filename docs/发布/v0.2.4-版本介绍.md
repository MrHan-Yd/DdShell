# DdShell v0.2.4 发布

DdShell v0.2.4 重点补齐了应用内更新链路。macOS 和 Windows 现在接入 Tauri 官方 updater，支持更新包签名校验、应用内下载与安装，并在安装完成后明确提示用户确认重启；Linux 暂时继续保留 GitHub Releases 下载页作为回退。

## 亮点

### macOS 和 Windows 接入官方应用内更新

- 状态栏和「设置 > 关于」现在共用同一套更新状态
- 更新包下载和安装走 Tauri 官方 updater
- 安装完成后只提示用户重启，不会自动重启应用
- 如果当前仍有连接或传输任务，重启确认会提示可能中断

### 更新过程更清晰

- 下载时显示进度
- 弱网或长时间无进度时显示慢网络提示
- 检查、下载、安装失败时保留打开 GitHub Releases 下载页的回退入口
- 当前最新 Release 暂无 `latest.json` 时，会回退到旧的 GitHub Releases 检查逻辑

### 发布链路补齐 updater 产物

- Release workflow 会上传 macOS `.app.tar.gz` 和 `.sig`
- Windows updater 使用 NSIS `.exe` 和 `.exe.sig`
- CI 会生成并上传 `latest.json`
- 普通安装包仍保留，方便手动下载安装

## 其他改进

- 补充发布文档，明确 updater signing secrets、manifest、产物命名和验证流程
- 补充 Tauri updater 发布契约，确保 Windows manifest 指向已签名的 NSIS `.exe`
- 继续保留 Linux GitHub Releases fallback

## 已知限制

- 这是包含官方 updater 的首个版本；已安装旧版本的用户升级到 v0.2.4 时，可能仍需要通过原有下载页或手动安装包完成。安装 v0.2.4 后，后续版本即可使用官方应用内更新。
- 正式发布前需要确认 GitHub Secrets 中的 updater 私钥和应用内置公钥匹配。
