# DdShell v0.2.5 发布

DdShell v0.2.5 重点增强终端内的智能辅助与文件管理能力。终端页现在提供可配置的 AI 助手、多模型切换、流式响应、推理信息展示和按服务器隔离的历史问答；同时新增终端底部文件管理抽屉，让 SSH 会话中的常用文件操作不必切换到独立 SFTP 页面。

## 亮点

### 终端 AI 助手

- 支持 OpenAI Chat Completions、OpenAI Responses、Claude Messages、Gemini Generate Content 和兼容接口
- 支持多个模型配置，并可在终端 AI 助手中切换配置与模型
- 支持流式响应状态、响应内容预览和可选推理信息展示
- 支持命令建议区分“方案”和“步骤”，多步骤命令可按顺序逐条执行
- 支持执行前二次确认开关，用户可选择直接执行或只插入命令等待手动回车
- 历史问答按服务器隔离保存最近 20 条，重新打开同一服务器仍可查看

### 终端内文件管理

- 新增终端底部文件管理抽屉，可在当前 SSH 会话中浏览、上传、下载、删除、重命名和新建目录
- 支持拖拽调整文件管理抽屉高度，并优化拖拽过程中的终端重排手感
- 支持文件覆盖预检查，上传或下载同名文件前会统一确认
- 支持快速移动文件到远程目录，并保留目录选择弹窗

### 终端工具栏与命令宏体验

- 右侧终端工具栏整合收藏、命令宏、AI 助手和文件管理入口
- 命令宏与 AI 助手互斥打开，避免多个浮层同时遮挡终端
- 命令宏弹层增加出现过渡动画、边框和紧凑布局
- 优化收藏、历史、命令宏等终端浮层层级，减少被工具栏遮挡的问题

## 其他改进

- 默认启用实时候选模式，并优化命令助手列表模式与右键菜单 hover 过渡
- 应用内更新下载过程增加进度展示和慢网络提示
- 补充 AI Agent provider 配置契约和发布 updater 契约，明确多模型、流式、推理信息和 Windows NSIS `.exe` updater 产物要求
- 修复终端文件管理多选下载时多个覆盖确认互相取消的问题
- 修复 AI 模型数值配置可保存为 0 导致 provider 请求失败的问题
- 发布版本号同步更新到 `0.2.5`

## 已知限制

- 自动拉取模型列表仍未实现，模型 ID 需要用户在设置中手动填写。
- Linux 暂时继续使用 GitHub Releases 下载页作为应用内更新回退路径。
