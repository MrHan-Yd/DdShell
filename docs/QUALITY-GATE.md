# Quality Gate

本文件定义“合并前必须满足”的硬门禁。

## 必须通过（全部满足）
1. 需求范围符合 `PRD` 与 `FR-INDEX`。
2. 未引入非目标功能（插件机制、内置远端编辑器）。
3. 接口与错误码符合 `TECH-SPEC` 与 `API-CONTRACTS`。
4. UI 交互符合 `UI-SPEC` 及专项规范。
5. 测试覆盖正常/异常/边界场景。
6. Ubuntu/CentOS 差异场景已验证（若任务相关）。
7. 安全策略未退化（凭据、指纹、日志脱敏）。
8. 文档已同步更新（PRD/测试/变更记录）。
9. `ACCEPTANCE-TEMPLATE` 验收结果为“通过”。
10. `CHANGELOG` 已记录本次变更。

## 阻断规则
- 任何一条未通过，禁止合并。
- 出现阻断问题必须附回滚方案。
