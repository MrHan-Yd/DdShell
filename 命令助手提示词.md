## Shell App Snippets/Command Center：内置回填实现

### 1. 核心任务
实现 Shell App 内置 Snippets 候选提示与回填能力，主流程为：  
终端输入区输入 `//` -> 弹出内联候选提示 -> 仅回填到终端输入区（不执行）。  
在“设置 -> 终端设置”中提供 `//` 触发模式开关（默认关闭），例如输入 `//dep` 后弹出候选列表。

### 2. 触发与拦截规则
- 默认不做系统级全局键盘监听。
- `//` 仅在 Shell App 终端输入区生效，且需规避 URL/路径等常见输入。
- 触发位置不限：允许在任意位置输入 `//`，不要求行首或空白后。
- 触发时机：仅输入 `//` 不提示；`//` 后必须继续输入至少 1 个非空白字符才弹出候选（位置不限），每页最多展示匹配的 10 条。
- 输入法：IME 组合输入期间不触发；仅在 compositionend 后按最终文本判断。
- 排除规则：检测到 `://` 协议头、UNC 路径 `\\`，或 `//` 后紧跟 `/` 的路径写法时不触发。
- 插入仅回填，不发送回车。
- 插入后必须恢复终端输入焦点。
- 误触退出：`Esc` 立即关闭候选并保留原始输入。
- 回填行为：选中并回填时，替换当前的 `//` 触发前缀为目标命令；`Esc` 取消时保留 `//` 原样。
- 设置项（设置 -> 终端设置）：`//` 触发开关（默认关闭）。
- 设置项（设置 -> 终端设置）：提供“重置偏好”入口，用于清除本地排序权重。
- 设置项（设置 -> 终端设置）：回填确认键，默认 `Tab` 回填；可选改为 `Enter`。
- 提示策略：仅当用户切换为 `Enter` 时弹出风险提示并二次确认（回车可能误触执行），默认不展示提示。
- 约束：不支持“先输入命令再按快捷键弹出候选”；候选仅由 `//` 触发。
- 引导：首次创建片段时提示“可在 设置 -> 终端设置 开启 `//` 快捷触发”。
- 引导：当用户在终端输入 `//` 但开关关闭时，给一次性轻提示并提供一键开启入口。
- 翻页规则：匹配总数 <= 10 时不分页；匹配总数 > 10 时按 10 条/页分页，下翻加载下一页。

### 3. 数据与索引
- 片段数据来源包含两类并共同参与匹配：用户自定义命令片段（沿用现有片段功能与存储）；系统内置常用命令（按发行版区分，如 Ubuntu/CentOS）。系统内置命令不展示在命令片段页面，仅用于候选匹配。
- 内存维护 Trie/前缀索引用于快速匹配。
- 采用 LRU 缓存热点片段正文，降低内存占用。

### 3.1 系统感知匹配
- 连接建立后识别远端系统类型（Ubuntu/CentOS 等）。
- 候选提示优先展示与当前系统匹配的内置命令。
- 用户片段与系统内置命令共同参与排序与过滤。

### 3.2 用户偏好排序
- 使用“权重”而非简单计数，体现最近选择更重要。
- 更新规则示例：每次回车回填时，`score = score * 0.9 + 1`（衰减系数可配置）。
- 说明：衰减会逐步降低历史影响，`score` 不会长期无限增长，可选增加硬上限。
- 归一化：对当前候选集合做归一化，`weight% = score / sum(scores) * 100`，候选权重总和恒为 100%。
- 排序：同一前缀命中的候选按 `weight%` 从高到低排序；权重相同再按系统匹配/前缀长度等基础规则排序。
- 权重数据存储在本地 SQLite，不展示于命令片段页面。

### 4. 交互与状态
- UI 行为遵循 `docs/03-UX/COMMAND-CENTER-SPEC.md`。
- UI 行为遵循 `docs/03-UX/TOKEN-INSERT-SPEC.md`。
- UI 行为遵循 `docs/03-UX/UI-SPEC.md`。
- 状态必须覆盖：加载、空态、错误态、成功回填。
- 长文本回填需确认（可关闭）。

### 5. 性能与资源约束
- 目标：在 10k 级片段下保持可感知无卡顿的检索与滚动体验。
- 预取策略：初始加载 10 条；当选中位置接近已加载批次的 70%~80%（如第 7~8 条）时，预取下一批 10 条（分页模式）。
- 索引与缓存容量需可配置，避免常驻内存膨胀。

### 6. 技术细节
- Rust：使用异步 I/O 读取片段正文（如 `tokio::fs`），确保不阻塞 UI。
- React：使用现有 UI 栈与 CSS 过渡实现轻量动画，避免新增依赖。
- 约束：不新增第三方 UI/动画库，沿用现有前端栈实现交互与动效。
- 数据流：Rust 进行匹配 -> React 展示候选 -> 选中后回填 -> Rust 预取正文。
