# Shell App PRD（个人版）

## 1. 文档信息
- 产品名称：Shell App（替代 FinalShell 的个人跨平台客户端）
- 文档版本：v1.0
- 目标发布日期：首版 6~8 周内完成
- 适用平台：Windows / macOS / Linux

## 2. 产品目标
- 提供高质感、低门槛、稳定的 SSH/SFTP 桌面客户端。
- 在个人使用场景下，体验接近或优于 FinalShell。
- 优先保证连接稳定性、终端效率和文件传输效率。

## 3. 目标用户与场景
- 目标用户：个人开发者、运维工程师、学生/独立开发者。
- 典型场景：
  - 日常连接多台服务器执行命令。
  - 通过 SFTP 上传/下载项目与日志。
  - 维护常用命令片段（Snippets）提高操作效率。

## 4. 范围定义
### 4.1 MVP（首版必须）
- 连接管理：新增/编辑/删除/搜索连接，支持分组。
- SSH 终端：多标签、基本分屏、会话重连。
- SFTP 双栏：浏览、上传、下载、重命名、删除。
- 设置中心：主题、字体、快捷键基础配置。
- 安全基础：凭据加密存储（keyring），known_hosts 首次确认。

### 4.2 v2（后续增强）
- 断点续传、传输队列持久化增强。
- 端口转发、代理配置高级项。
- 会话快照、更多自动化能力。

### 4.3 非目标
- 团队协作与权限审批。
- 云账号体系与多端实时同步。
- 插件市场。

## 5. 功能需求与验收标准

### FR-01 连接管理
**描述**
- 用户可创建、编辑、删除 SSH 连接配置。
- 支持按名称/主机快速搜索。

**验收标准**
- 新建连接后，连接项立即出现在列表。
- 编辑连接后，列表与详情实时更新。
- 删除连接后，不可在列表与搜索结果中出现。
- 搜索关键词后，200ms 内返回过滤结果（50 条配置量级）。

### FR-02 SSH 会话
**描述**
- 用户可发起 SSH 连接并在终端中操作。
- 支持多标签、标签切换、关闭当前标签。

**验收标准**
- 连接成功后，2.5s 内进入可输入终端（冷启动场景）。
- 同时打开 5 个会话，切换标签操作无明显卡顿。
- 断线后可手动重连，并恢复可用输入状态。

### FR-03 终端交互
**描述**
- 提供复制、粘贴、清屏、分屏等高频能力。

**验收标准**
- 粘贴大段命令不丢字、不乱序。
- 分屏后各终端可独立输入与滚动。
- 快捷键可触发核心动作（新建/关闭/切换标签）。

### FR-04 SFTP 文件管理
**描述**
- 本地/远程双栏浏览与基础文件操作。

**验收标准**
- 可打开目录并展示文件列表。
- 上传/下载任务可显示进度、速度、状态。
- 对常见失败（权限不足、网络中断）展示可理解错误。

### FR-05 Snippets
**描述**
- 用户可维护常用命令片段并一键插入终端。

**验收标准**
- 可新增、编辑、删除片段。
- 片段插入行为在当前活动终端生效。

### FR-06 设置与个性化
**描述**
- 支持主题、字体字号、终端基础偏好设置。

**验收标准**
- 设置保存后重启应用仍生效。
- 主题切换时界面元素无明显错位。

### FR-07 安全与数据存储
**描述**
- 密码与私钥口令不以明文写入数据库。

**验收标准**
- 凭据通过系统 keyring 读写。
- 首次连接主机时触发指纹确认。
- 本地日志默认脱敏主机和用户名。

## 6. 非功能需求
- 性能：
  - 冷启动 < 2.5s。
  - 常驻内存目标 < 250MB（普通使用）。
  - 1000 文件目录渲染 < 300ms。
- 稳定性：
  - 网络波动下连接自动重试（指数退避）。
  - 传输异常后任务状态可追踪。
- 可用性：
  - 支持深色主题。
  - 快捷键与鼠标操作并行可用。

## 7. 关键流程
- 新建连接 -> 测试连接 -> 保存 -> 发起会话。
- 进入会话 -> 执行命令 -> 断线 -> 重连恢复。
- 打开 SFTP -> 浏览目录 -> 上传/下载 -> 查看结果。

## 8. 里程碑与交付物
- M1（第 1~2 周）：工程骨架 + 主题系统 + 存储基础。
- M2（第 3~4 周）：SSH 会话链路 + 连接 CRUD。
- M3（第 5~6 周）：SFTP 双栏 + 快捷键 + Snippets。
- M4（第 7~8 周）：稳定性打磨 + 打包发布。

## 9. 验收清单（DoD）
- 三平台可安装并稳定启动。
- 支持 >= 50 条连接配置并可顺畅搜索。
- 多标签 SSH 会话可稳定使用。
- SFTP 上传/下载在常见网络环境可用。
- 敏感信息不明文落盘。
- UI 样式符合统一规范（间距/状态/动效一致）。

## 10. 风险与缓解
- Linux 桌面差异：提供视觉降级策略。
- SSH 库行为差异：MVP 优先 `ssh2`，后续评估迁移。
- 过度设计风险：以“信息密度 + 可读性”为第一优先级。
